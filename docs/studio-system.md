# Studio 体系架构

## 面向对象
**Studio:** 拥有专门属性来执行某一类问题的子 agent 集合，比如 "策划师", "前端工程师", "后端工程师"。

## 数据结构
- **Card**: 用来供 Agent Zero 快速路由匹配，不含上下文，仅包含能力标签
- **Soul**: （本阶段以全量拼接为主，后续可做时间线分层记忆摘要压缩）
- **Agent.md**: 系统预设或用户修改的角色要求（例如："你是一个资深React前端专家..."）

## 何时实例化？
只有在 0 号分析完当前任务类型并确认对应 Studio 能力要求时，Studio 才被“调参”生成子节点并在画布中映射执行环节。
