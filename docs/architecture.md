# 整体架构设计

AStudio 采用三层架构，从逻辑到展现完全解耦。

## 0 号 Agent (CEO)
处理全局路由、拆解、汇总：
- `/api/ask` 接口统一接收问题
- Pydantic 模型作为输入输出标准
- 用 LiteLLM 做模型调度

## Studio (部门/工作室)
面向场景的问题解决专家：
- `agent.md`：核心定义，用户可改
- `soul.md`：运行时蒸馏积累的记忆
- 被 0 号动态组织起来执行 Task

## Canvas (作战室)
空间化可视的执行流：
- `task`: 任务实例
- `node`: 某一次具体的思考/询问节点
- `edge`: 父子关系、发散关系
- 实时利用 SSE 向前端下发执行状态
